# Labs — Summer 2026 submission guide

Create one folder for each published lab using the exact roadmap key:

```text
labs/
├── lab-01/
│   ├── index.html
│   ├── source and assets
│   └── README.md
└── lab-XX/
```

For each lab:

1. Follow the linked course document; it overrides this general guide.
2. Keep every file needed to run or review the lab inside `labs/<lab-key>/`.
3. Add a short `README.md` explaining how to run the work, what you completed, and any known limitations.
4. Use relative paths so assets work locally and through GitHub Pages.
5. Commit and push, then update the matching GitHub Issue's checklist and **Student submission** section.
6. Add `ready-for-approval` when the lab is ready to be graded.

Do not rename the lab folder or key label. The stable key connects the source folder, Issue, milestone, and automated status table.
